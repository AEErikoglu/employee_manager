## Development Tipps
- Use a clearly laid out State Management for the data from backend
- Use a clear folder structure for further Implementation
- Use comments for documentational purposes after implementing a core function
- Do not use Mock data if no data from backend is available at the moment
- This app doesn't contain any scrollable view
- Use a reactive design technic to fit the app in all possible phone displays
- Implement every view in its own xaml file
- Use the Font type INTER

## Color Palette
- #0B1B32 → page background
- #0D1E4C → card / phone frame
- #26415E → input surfaces and secondary elements
- #C48CB3 → primary accent (buttons, links, focus rings)
- #83A6CE → secondary text and icons
- #E5C9D7 → main foreground text
